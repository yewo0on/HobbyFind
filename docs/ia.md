# IA — HobbyFind

## 1. 전체 사이트맵 구조 (트리 형태)

### 1.1 상위 IA 트리

```
/ (홈)
├─ Hero 섹션
├─ 대분류 필터 토글 (운동형 / 지능형 / 예술형)
└─ 취미 카드 그리드 (전체 or 필터 적용)

/category/[type]
├─ /category/sports (운동형)
├─ /category/intelligence (지능형)
└─ /category/art (예술형)

/login
/signup

/mypage (로그인 필요)
```

### 1.2 URL 기반 트리

* **/**
  * 접근 권한: 비회원/회원 공통
* **/category**
  * `/category/sports`
  * `/category/intelligence`
  * `/category/art`
* **/auth**
  * `/login`
  * `/signup`
* **/mypage**
  * 접근 권한: 로그인 사용자만

(Next.js 기준: `app/page.tsx`, `app/category/[type]/page.tsx`, `app/login/page.tsx`, `app/signup/page.tsx`, `app/mypage/page.tsx` 등으로 구현 가능)

## 2. 사용자 흐름 (User Flow)

### 2.1 비회원 플로우

**탐색 중심**

1. 비회원이 `/` 접속
2. Hero 섹션 및 전체 취미 카드 그리드 확인
3. 상단바 카테고리 메뉴에서
   * `운동형` 클릭 → `/category/sports`
   * `지능형` 클릭 → `/category/intelligence`
   * `예술형` 클릭 → `/category/art`
4. 각 카테고리 페이지에서 취미 카드 상세 정보를 탐색
5. 북마크 버튼을 보더라도,
   * 클릭 시 로그인 필요 안내 (툴팁/간단 메시지)
6. 상단바의 **로그인** 또는 **회원가입** 클릭 시
   * `/login` 또는 `/signup`으로 이동

### 2.2 회원 플로우

**탐색 + 북마크 + 마이페이지**

1. 비회원이 `/login` 또는 `/signup`으로 진입
2. 회원가입 성공 → 자동 로그인 또는 로그인 페이지로 이동 후 로그인
3. 로그인 성공 후
   * `/` 또는 직전 페이지로 리다이렉트
   * 상단바: 로그인/회원가입 → 마이페이지/로그아웃으로 변경
4. 홈/카테고리 페이지에서 관심 취미 카드의 **북마크 버튼 토글**
5. 상단바 **마이페이지** 클릭 → `/mypage` 이동
6. `/mypage`에서
   * 북마크 리스트 확인
   * 북마크 총 개수 및 카테고리별 분포 통계 차트 확인
7. 필요 시 마이페이지 내 또는 카드에서 북마크 해제
8. 상단바 **로그아웃** 클릭 → 세션 종료, 비회원 상태로 전환

## 4. 페이지 계층 구조 (Page Hierarchy)

### 4.1 홈 (`/`)

* **Top Bar**
* **Hero 섹션**
* **메인 콘텐츠**
  * 대분류 필터 토글 (운동형·지능형·예술형)
  * 취미 카드 그리드

### 4.2 카테고리별 페이지 (`/category/[type]`)

* **Top Bar**
* **카테고리 헤더**
  * 선택된 카테고리명
  * 카테고리 소개 문구
* **카드 그리드 영역**
  * 해당 카테고리 취미 카드 그리드

### 4.3 로그인 (`/login`)

* **Top Bar**
* **로그인 폼 섹션**
  * 아이디 입력
  * 비밀번호 입력
  * 로그인 버튼
  * 에러 메시지 영역
  * 회원가입 링크

### 4.4 회원가입 (`/signup`)

* **Top Bar**
* **회원가입 폼 섹션**
  * 아이디 입력
  * 비밀번호 입력
  * 약관 동의 체크박스
  * 회원가입 버튼
  * 로그인 링크

### 4.5 마이페이지 (`/mypage`)

* **Top Bar**
* **요약 헤더 섹션**
  * "마이페이지" 타이틀
  * 북마크 총 개수 요약 텍스트
* **본문 영역**
  * 북마크 목록 (리스트/그리드)
  * 카테고리별 분포 차트 섹션

## 6. 상호작용 패턴 (Interaction Patterns)

| 상호작용 | 트리거 | 상태 변화 / 결과 |
| --- | --- | --- |
| 카테고리 메뉴 클릭 | Top Bar의 "운동형/지능형/예술형" 클릭 | `/category/[type]`로 라우팅, 해당 카테고리 페이지 렌더 |
| 홈 내 대분류 필터 토글 | 홈 메인 섹션의 필터 토글 클릭 | 홈 페이지 내 그리드에서 해당 카테고리만 필터링 (URL은 `/` 유지) |
| 북마크 토글 (회원) | 카드의 북마크 버튼 클릭 | 활성/비활성 상태 토글, DB/스토리지 업데이트, 마이페이지 통계/리스트에 반영 |
| 북마크 시도 (비회원) | 카드의 북마크 버튼 클릭 | 북마크 미적용, "로그인이 필요합니다" 등 안내 메시지 |
| 로그인 시도 | `/login`에서 폼 제출 | 성공 시 세션 생성 및 리다이렉트, Top Bar가 로그인 상태용으로 변경 |
| 로그인 에러 | 잘못된 아이디/비밀번호로 폼 제출 | 에러 메시지 영역에 문구 표시 |
| 회원가입 시도 | `/signup`에서 폼 제출 | 성공 시 로그인 또는 로그인 페이지로 이동, 에러 시 폼 내 메시지 표시 |
| 마이페이지 진입 | Top Bar의 마이페이지 클릭 | 로그인 상태면 `/mypage` 진입, 비로그인 상태면 `/login`으로 리다이렉트 |
| 로그아웃 | Top Bar의 로그아웃 클릭 | 세션 삭제, `/` 이동, Top Bar가 비로그인 상태용으로 변경 |

## 10. 기술 스택 고려 (Next.js 기반 IA 반영)

* **라우팅**: Next.js App Router의 파일 기반 라우팅 활용 (`app/` 디렉토리)
* **레이아웃**: `app/layout.tsx`에서 `TopBar` 및 글로벌 스타일 설정
* **페이지 컴포넌트 구조**: 각 URL에 대응하는 `page.tsx` 파일에서 위 IA의 섹션/컴포넌트 계층을 구현
* **상태 관리 & 상호작용**: 북마크 토글, 로그인 상태, 필터 토글 등은 클라이언트 컴포넌트와 서버 액션/간단 API를 조합해 구현
* **정적 데이터**: 취미 리스트는 상수/JSON로 고정 관리 (운동형·지능형·예술형 세트, 확장 없음)

이 IA를 그대로 반영하면, 현재 디자인/코드 구조(Top Bar, 카테고리 페이지, 마이페이지 등)를 Next.js 기준으로 일관성 있게 설계·구현할 수 있습니다.
